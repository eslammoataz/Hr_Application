**Branch**: `main` | **Date**: 2026-04-13 | **Spec**: [spec.md](./spec.md)

## Summary

Nine targeted refactor tasks to clean up constitution violations in the live HRMS codebase. No new features, no API changes, no behavioral changes — only structural improvements: remove dead code, add type safety, optimize read-only queries, and extract a large handler into a service. All changes are backend-only; Flutter mobile app is unaffected.

## Phase 0: Research & Decisions

*Not applicable.* This is a refactor of existing code with explicit task specifications. All technical decisions are already determined in `spec.md`:

| Decision | Rationale |
|----------|-----------|
| Delete `Domain/Common/BaseEntity.cs` | Shadowing `Domain/Models/BaseEntity.cs`; has no `Id` or `IsDeleted` |
| Add `HasQueryFilter` to `Request` | All other entities have it; `Request` was the only exception |
| Use `.HasConversion<string>()` for enum | DB stores same string values — no migration needed |
| Use `.AsNoTracking()` on 11 methods | Read-only queries; EF tracking is unnecessary overhead |
| Keep 3 `SaveChangesAsync` in ClockIn | FK constraints require attendance.Id before log can reference it |
| Mapster register over inline `.Select()` | Reusable, testable; output must match manual projection exactly |
